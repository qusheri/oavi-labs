## 1. Цветовые модели

### 1.1. Выделение компонент R, G, B
**Используемый метод:**  
Исходное изображение в модели RGB представляется тремя каналами: красным (**R**), зелёным (**G**) и синим (**B**). Для выделения компонент из массива изображения отдельно сохранялись значения каждого канала как полутоновые изображения.

Формально:
- `R(x, y) = img[x, y, 0]`
- `G(x, y) = img[x, y, 1]`
- `B(x, y) = img[x, y, 2]`

### 1.2. Преобразование RGB → HSI и выделение яркостной компоненты
**Используемый метод:**  
Изображение переводилось из модели RGB в модель **HSI** (**Hue, Saturation, Intensity**), где:
- **H** — цветовой тон,
- **S** — насыщенность,
- **I** — яркость.

Яркостная компонента вычислялась по формуле:

```text
I = (R + G + B) / 3
```

где `R`, `G`, `B` предварительно нормировались к диапазону `[0, 1]`.

#### Результат
![HSI Intensity](../lab_output/4_HSI_intensity.png)

---

### 1.3. Инверсия яркостной компоненты в исходном изображении
**Используемый метод:**  
После перевода в модель HSI яркостная компонента инвертировалась:

```text
I' = 1 - I
```

После этого изображение переводилось обратно в RGB при сохранении исходных значений **H** и **S**.

#### Результат
![Inverted intensity](../lab_output/5_inverted_intensity_rgb.png)

**Вывод:** инверсия яркости изменила светлоту исходного изображения, сохранив при этом оттенки и насыщенность настолько, насколько это допускает обратное преобразование HSI → RGB.

## 2. Передискретизация изображений

### Общий принцип
Для изменения размеров изображения использована **ручная билинейная интерполяция**, без применения встроенных библиотечных функций ресемплинга.

**Используемый метод:** билинейная интерполяция.  
Для каждого пикселя результирующего изображения вычислялись координаты в исходном изображении, после чего значение определялось как взвешенное среднее четырёх ближайших пикселей.

Для точки с дробными координатами `(x, y)`:

```text
P(x, y) = (1-wy) * ((1-wx) * P(x0, y0) + wx * P(x1, y0))
        + wy     * ((1-wx) * P(x0, y1) + wx * P(x1, y1))
```

где:
- `x0 = floor(x)`, `x1 = x0 + 1`
- `y0 = floor(y)`, `y1 = y0 + 1`
- `wx`, `wy` — дробные части координат.

В примере использованы коэффициенты:
- `M = 2` — растяжение в 2 раза,
- `N = 3` — сжатие в 3 раза,
- `K = M / N = 2 / 3`.

---

### 2.1. Растяжение изображения в M раз
**Используемый метод:**  
Размер изображения увеличивался в `M = 2` раза по каждой координате с помощью билинейной интерполяции.

#### Результат
![Stretch x2](../lab_output/6_stretch_x2.png)

---

### 2.2. Сжатие изображения в N раз
**Используемый метод:**  
Размер изображения уменьшался в `N = 3` раза по каждой координате. Значения новых пикселей вычислялись той же билинейной схемой, но уже при отображении координат из уменьшенного изображения в исходное.

#### Результат
![Compress x1_3](../lab_output/7_compress_x1_3.png)

---

### 2.3. Передискретизация в коэффициент K = M/N в два прохода
**Используемый метод:**  
Передискретизация выполнялась в два этапа:
1. сначала изображение растягивалось в `M = 2` раза;
2. затем результат сжимался в `N = 3` раза.

Итоговый коэффициент масштабирования:

```text
K = M / N = 2 / 3
```

#### Результат
![Two pass](../lab_output/8_resample_two_pass_x2_3.png)

---

### 2.4. Передискретизация в коэффициент K за один проход
**Используемый метод:**  
Изменение размера выполнялось сразу с коэффициентом:

```text
K = 2 / 3
```

без промежуточного этапа увеличения изображения.

#### Результат
![One pass](../lab_output/9_resample_one_pass_x2_3.png)

## Сравнение результатов
- При выделении каналов RGB видно, как различается вклад каждого цветового канала в исходное изображение.
- Яркостная компонента HSI позволяет отделить информацию о светлоте от информации о цвете.
- Инверсия яркости приводит к изменению визуального восприятия изображения при сохранении цветовой структуры.
- Растяжение увеличивает размер изображения, но при этом может приводить к размытию мелких деталей.
- Сжатие уменьшает размер изображения и частично теряет высокочастотные детали.
- Передискретизация в два прохода и за один проход с одинаковым итоговым коэффициентом может давать слегка различающийся результат из-за промежуточной интерполяции.
